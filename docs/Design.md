# 디자인 철학

이 문서는 Weatherpane의 시각적 정체성을 정의한다. 모든 UI 결정의 기준이 된다.

## 창의적 방향: 대기의 편집자 (The Atmospheric Editor)

날씨 데이터를 고급 여행 에디토리얼처럼 다룬다. Airbnb의 따뜻함과 한국 미니멀리즘의 정밀함을 결합한다.
사진 중심의 레이아웃, 의도적인 비대칭, 극적인 타이포그래피 스케일로 "템플릿 느낌"을 탈피한다.

---

## 테마 시스템

| 테마 | 이름 | 분위기 |
|---|---|---|
| 밝은 모드 | 햇살 (Haet-Ssal) | 따뜻한 한지, 편집 갤러리 |
| 어두운 모드 | 달빛 밤 (Dal-Bit Night) | 심층적 몰입, 영화적 야경 |

테마 전환: `<html>` 요소에 `.dark` 클래스 적용/제거로 제어한다.
첫 방문 시 시스템 환경설정을 따른다. 이후 명시적 선택을 유지한다.

## 색상 철학

### 핵심 원칙: "선 없음" 규칙
1px 실선 테두리로 영역을 구분하는 것은 **금지**한다. 경계는 다음 방법으로만 표현한다:
1. **배경색 전환**: `surface-container-low` 카드를 `background` 위에 배치
2. **색조 전환**: 은은한 색상 이동으로 포함 관계 암시
3. **네거티브 스페이스**: 간격만으로 그룹화

### 표면 계층 (밝은 모드)
```
background (#fcf9f8)                    ← 전체 페이지 기반
  └─ muted (#f0eded)                    ← 보조 섹션
      └─ card (#ffffff)                 ← 우선 카드, 팝오버
```

### 표면 계층 (어두운 모드)
```
background (#131313)                    ← 전체 페이지 기반
  └─ muted (#20201f)                    ← 보조 섹션
      └─ card (#1c1b1b)                 ← 카드
          └─ popover (#20201f)          ← 팝오버, 부유 요소
```

### 주요 색상: Rausch Red
- 밝은 모드: `primary = #ba0036` (진한 빨강)
- 어두운 모드: `primary = #ffb2b6` (밝은 분홍빨강, 어두운 배경 가독성)
- **절제 원칙**: primary는 활성 상태, 긴급 날씨 경보에만 사용한다. 과용 시 임팩트 소멸.

### 보조 색상: 에메랄드
- tertiary 계열은 기상 상태 표시 (좋음/위험)에 사용한다.
- `tertiary = #006a45` (밝은) / `#62dca3` (어두운)

## 타이포그래피

| 역할 | 밝은 모드 | 어두운 모드 |
|---|---|---|
| 디스플레이/헤드라인 | Plus Jakarta Sans | Plus Jakarta Sans |
| 본문/레이블 | Be Vietnam Pro | Plus Jakarta Sans |

타입 계층 원칙:
- `display-lg`: 현재 기온, 도시명 (화면 상단 40% 점유)
- `headline`: 섹션 앵커
- `body`: 데이터 수치, 설명
- 크기 간격을 크게 유지한다 — 중간 크기는 위계를 흐린다

## 형태 언어

### 반경 스케일

| 토큰 | 값 | 적용 대상 |
|---|---|---|
| `--radius-sm` | `0.5rem (8px)` | 버튼 |
| `--radius-md` | `1.5rem (24px)` | 카드, 에디토리얼 타일 |
| `--radius-lg` | `2rem (32px)` | 모달, 대형 패널 |
| `--radius-full` | `9999px` | 칩, 필터, 알약형 UI |

### 유리형태 (Glassmorphism)
부유 요소(알림, "지금" 위젯)에 사용:
- 밝은 모드: `surface-container-highest` 60% 불투명도 + `backdrop-blur: 20px`
- 어두운 모드: `surface-bright (#393939)` 40% 불투명도 + `backdrop-blur: 20px`

## 그림자 시스템

"떠있는" 요소(날씨 경보, 모달)에만 그림자를 사용한다. 평면 카드에는 사용하지 않는다.

세 겹 앰비언트 시스템 (`--shadow-float`):
- Layer 1: `0px 2px 4px` — 4% 불투명도
- Layer 2: `0px 4px 12px` — 4% 불투명도
- Layer 3: `0px 10px 24px` — 4% 불투명도

그림자 색상은 순수 회색이 아닌 `on-surface` 기반 색조를 사용한다.

## 금지 사항 요약

| 금지 | 이유 |
|---|---|
| 1px 실선 테두리 | 프리미엄 에디토리얼 느낌 파괴 |
| 순수 검정 (#000000) | 유리 효과 소멸, 눈 피로 |
| 파란색/노란색 날씨 아이콘 | 브랜드 일관성 위반 |
| 평면 카드에 그림자 | 색상 전환으로 깊이 표현 충분 |
| 100% 불투명 구분선 | 네거티브 스페이스 또는 15% 불투명도 사용 |
